* Performance convention: Prefer native Bash string manipulation (e.g., parameter expansion like `${var//./ }` and `extglob`) over spawning external binaries (like `sed`, `awk`, or `basename`) inside subshells when operating within busy loops or frequently called functions to significantly reduce overhead.
* Performance convention: Prefer native Bash string manipulation (e.g., parameter expansion like `${var#"${var%%[!0]*}"}`) over spawning external binaries (like `sed`) inside subshells to significantly reduce overhead for tasks like removing leading zeros.
